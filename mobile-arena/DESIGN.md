# Wave Arena — 要件定義（モバイルアプリ主軸）

本書は `mobile-arena`（静的PWA / Capacitor ネイティブ）の要件定義。
親プロジェクト `demo0902`（Java サーブレット版）の要件 `../DESIGN.md` を土台に、
**モバイルアプリを主軸**としたうえで**PWA デスクトップ版も併存**させる方針へ更新したもの。

> 位置づけ: 「PCゲームをモバイルでも動くようにした」状態から、
> 「モバイルを第一級プラットフォームとし、デスクトップも対応する」状態へ反転させる。

---

## 0. プラットフォーム方針

- **主軸**: モバイル（タッチ）。スマホ縦/横持ち、タブレットを第一に設計する。
- **併存（Windows版）**: PWA デスクトップ版（キーボード＋マウス）。機能を削らず維持する。
- **配布形態**: ①PWA（ブラウザ/ホーム画面）②Capacitor ネイティブ（iOS/Android）。
- サーバ不要（スコア/セーブはローカル永続。ネイティブでは Preferences ミラー）。

---

## 1. 既存要件（親 DESIGN.md から継承・実装済み）

以下は `../DESIGN.md` 3〜10章を継承する。mobile-arena では実装済みであり、本書では
**変更・追加のある点のみ**後続章で上書きする。

- ゲームループ: ウェーブ制ローグライト（殲滅→強化カード3択→次波、中ボス/ボス）。
- データ駆動の敵・攻撃（`core/config.js`）、開発者モード（パラメータエディタ）。
- 演出: カメラ追従・画面シェイク・FX・補間描画・被弾方向インジケータ・ボス撃破スロー等。
- タッチ操作: ツインスティック（アナログ移動／照準＋射撃）、操作設定（左右入替・透明度・
  サイズ・自動射撃）、横向き推奨ヒント、AudioContext の初回タップ resume。
- 永続化: スコアランキング（到達WAVE優先）＋スロットセーブ（`localStorage`、`arena_` 接頭辞）。
- PWA: `manifest.webmanifest` ＋ `sw.js`（相対パス・cache-first）。
- 単体テスト（`node --test`）＋ CI、GitHub Pages デプロイ。
- Capacitor 対応（`capacitor.config.json` / `scripts/build-www.mjs` / `js/services/kv.js`）。

## 2. 新要件（モバイル主軸化で満たすべきもの）

> 凡例: ✅=実装済み / ⬜=未実装（後続フェーズで実装）

### 2.1 画面適応（フェーズ1）
- ✅ **カメラズーム**: 画面サイズ/DPR に依らず、可視ワールドの縦幅を一定（既定 ~15 タイル）に
  保つ。縦長スマホでスプライトが極小化する問題を解消する（`renderFrame` の `ctx.scale`）。
  - マップが画面に収まる場合ははみ出さない最小ズーム（fitZoom）以上を保つ。
  - ズーム量は `state.viewZoom`、可視範囲は `state.viewW/viewH`、カメラ原点は `state.camX/camY`
    として保持し、入力のスクリーン→ワールド座標変換に使えるようにする。
- ⬜ **セーフエリア対応**: ノッチ/ホームインジケータを避けて HUD・タッチUI・オーバーレイを
  配置（`env(safe-area-inset-*)`）。

### 2.2 タッチで完結する操作系（フェーズ2）
要件: **キーボード非接続の端末だけで全機能に到達できること**。キー専用機能を撤廃する。
- ⬜ セーブ/ロードをタッチUIから起動できる（現状は P/L キー専用 → タッチUIのボタン/メニュー化）。
- ⬜ ポーズ時に操作できる**ポーズメニュー**（再開/セーブ/ロード/設定/タイトル相当）を画面UIで提供。
- ⬜ 既存のタッチ設定（⚙）に上記の導線を統合、もしくはポーズメニューに集約。
- 受入基準: タッチのみで「開始→プレイ→ポーズ→セーブ→ロード→設定変更→ゲームオーバー→
  リスタート」が一通り操作できる。

### 2.3 操作系のモバイル最適化（フェーズ3）
- ⬜ ツインスティックの追従/デッドゾーン/復帰の調整（誤操作低減）。
- ⬜ 自動照準（autoFire）の既定 ON をモバイルで選択可能に（設定で切替）。
- ⬜ 武器切替をタッチで素早く行える UI（ボタン巡回は実装済み。改善は任意）。
- 制約: ゲームロジック（`input.keys/aim/move/autoFire`）の契約は変えない。UI 層のみで対応。

### 2.4 ネイティブ機能統合（フェーズ4・Capacitor 前提）
- ⬜ **ハプティクス**: 被弾・ボス撃破等で軽い振動（`@capacitor/haptics`）。Web では無効。
- ⬜ **Android 戻るボタン**: ポーズ/メニューを閉じる、最上位ではアプリ終了確認。
- ⬜ **ステータスバー/スプラッシュ**: フルスクリーン・暗色テーマに整合。
- ⬜ **アプリ復帰/中断**: バックグラウンド化で自動ポーズ、復帰で再開（`App` プラグイン）。
- 制約: すべて**任意依存**（プラグイン不在の Web/PWA では no-op フォールバック）。

## 3. 非機能要件（モバイル観点の追加）

- 性能: 中位スマホで 60fps を目標（敵上限時も）。DPR は上限 2.5 にクランプ。
- 互換: モダンモバイルブラウザ（iOS Safari / Android Chrome）＋ Capacitor WebView。
- 耐久性: WebView の `localStorage` 消失に備え、ネイティブでは Preferences へミラー。
- 操作のしやすさ: タッチターゲットは十分な大きさ、セーフエリアを侵さない。
- 後方互換: デスクトップ（キーボード/マウス）操作とロジックを壊さない。

---

## 4. 実装フェーズ（依存順）

1. **画面適応の刷新**（✅ ズーム実装済み・本書で要件化 / ⬜ セーフエリア）。
2. **タッチUX一新**（セーブ/ロード/ポーズメニューの画面UI化）。
3. **操作系の最適化**（スティック/自動照準/武器切替）。
4. **ネイティブ統合**（ハプティクス/戻るボタン/ステータスバー/復帰時ポーズ）。

各フェーズで `node --test` を緑に保ち、ロジック層の契約（`input.*`）を不変に維持する。

## 5. 既知の制約

- ストア配布の APK/IPA ビルドには各 OS の SDK が必要（本リポジトリには生成物を含めない）。
- 認証は無し（実験用途・ローカル完結）。スコアは端末ローカルのみ。
