# 宇宙物理刷新（Space Physics Overhaul）設計書

- 日付: 2026-06-29
- ブランチ: `redesign/android-native-kotlin`
- ステータス: 設計確定（実装計画前）
- 位置づけ: 「Living Planets Expansion」の**最初の土台フェーズ**。本設計は物理の手触りに集中し、惑星・地表ステージ・生き物AI・会話などの大規模拡張は後続ラウンドに分割する。

> **後続更新（v2.16.0 / v2.17.0）**: 自滅多発のため壁・惑星への**激突／落下ダメージは廃止**（v2.16.0）。その後、宇宙は**完全摩擦ゼロの慣性移動**へ移行（v2.17.0）。本書 §3 の「ハイブリッド慣性」は通常移動に速度上限を残す形で踏襲しつつ、減衰はゼロ化。ダッシュを2系統に分割した（スティックを大きく倒す微加速の流しダッシュ＝スタミナ極小／ボタンの向き方向・強加速ダッシュ＝スタミナ大）。実装は純粋ロジック `sim/SpaceDrive.kt`（単体テスト付き）に集約。

## 1. 背景と目的

このゲームは見下ろし型・宇宙サバイバルARPG（v2.11.0）。プレイヤーの要望は「宇宙空間なので抵抗がなく、ダッシュで加速したらそのまま滑り続ける。惑星が重力を発し、ダッシュが姿勢制御・加速・重力脱出を担う。敵には重力を無視できる種など多様性がある」。

目的は、**現在のアーケード的な移動を、宇宙的な慣性＋重力の駆け引きへ刷新**すること。重力を「速度に効く力」にすることで、引かれる敵を惑星へ叩きつける、ダッシュで重力圏を振り切る、といった戦術を成立させる。

## 2. 現状分析（コードベース v2.11.0）

| 観点 | 望む姿 | 現状 |
|---|---|---|
| 慣性 | 真の momentum（離すと滑空、ごく僅かに減衰） | 摩擦＋速度上限モデル。`STOP_FRICTION=0.08` で離すとコンマ数秒で停止。慣性なし |
| ダッシュ | 姿勢制御＋加速＋重力脱出 | 速度上限と加速度を上げるのみ。重力との相互作用なし |
| `dashThrust` | 慣性の主パラメータ | **死にコード**。`PlayerConfig` に定義はあるが v2.9.0 の加速度モデル移行で `DASH_ACCEL` 定数に置換され未使用 |
| 重力源 | 惑星（離散天体） | 破壊可能な壁（小惑星）の塊のみ（`WallGravity`） |
| 重力の効果 | 速度に加算（軌道に落ちる） | **座標へ直接加算**（`t.x += a*dt`）。momentum に効かない |
| 重力の対象 | 種ごとに差（無視する敵あり） | プレイヤー・敵・弾を**一律**に引く |
| 衝突 | 高速衝突でダメージ（武器化／危険） | AABB対タイルのみ。クラッシュ概念なし |

### 活用できる既存資産
- `Velocity` は全エンティティが `driftX/driftY` を保持するが、**敵は未使用**（敵の `vx/vy` は高速減衰ノックバック専用）→ 敵 momentum に新コンポーネント不要。
- `Collision.moveAndCollide` は `hitX/hitY` を返す → 壁激突検知＝クラッシュ判定に流用可。
- `WallGravity.gravityAt` は純粋関数 → 惑星重力も同型の純粋関数で追加でき、テストが容易。

## 3. 設計判断サマリ

確定済みの主要判断（ブレスト合意）：

1. **対象範囲**: 宇宙物理の刷新に集中（Living Planets 全体は後続）。
2. **重力源**: 離散惑星（主）＋小惑星クラスタ（副）の併存。
3. **慣性モデル**: ハイブリッド（通常移動は操作権限を残し、ダッシュが強い持続 momentum と重力減衰を担う）。
4. **重力と敵**: 段階応答（データ駆動）＋衝突活用（高速衝突でダメージ＝重力の武器化、プレイヤーも被弾）。

既定値（要・実機チューニング、§6）：
- 弾/ビーム/敵弾は直進、**グレネードのみ**重力で曲がる。
- ダッシュ中の重力は **×0.25**（完全免疫ではない）。
- クラッシュは弾まず減速＋軽い反発、閾値超過分にダメージ比例。
- 惑星は宇宙ステージに **2〜4個**、半径・質量ランダム（seed 固定）。

## 4. アーキテクチャ

### 方針
既存システムを**その場で拡張**する（新モード分岐や並行物理レイヤは作らない）。計算ロジックは libGDX/Fleks 非依存の純粋モジュールに切り出して TDD し、ECS システムは薄く保つ（現行の流儀を踏襲）。

### 4.1 純粋モジュール（テスト対象の中心）
- `sim/Inertia.kt`（新規）: drift の積分・宇宙減衰・速度上限クランプ。プレイヤーと敵が共有する momentum 計算。
- `sim/PlanetGravity.kt`（新規）: 惑星リストからの重力加速度。`gravityAccelAt(planets, x, y, ...)`。
- `sim/WallGravity.kt`（既存）: `gravityAt` を併用（小惑星クラスタ）。両者を合算する `combinedGravityAccel(...)` を用意。
- `sim/CircleCollision.kt`（新規）: 円（惑星）対 AABB の押し出し＋内向き速度の取得。純粋関数。
- `sim/CrashModel.kt`（新規）: 内向き速度と閾値からダメージ量と反発後速度を算出。

### 4.2 コンポーネント
- `Velocity`（既存）: `driftX/driftY` を **momentum** として全エンティティで使用。`vx/vy` は従来どおりノックバック専用。変更なし（用途拡張のみ）。
- `Planet`（新規コンポーネント）: `radius: Float`, `mass: Float`, `biome: Biome`, `gravityRange: Float`。`Transform`（＋必要に応じ識別用タグ）と組み合わせる。将来 Living Planets の着陸対象になる土台。
- `EnemyDef`（既存・データ）: `gravityResponse: Float = 1f` を追加。`0`=重力無視 / `1`=通常 / `>1`=重い。

### 4.3 システム（既存を改修）
- `GravitySystem`: 効果を「座標加算」→「**drift 加算**」に変更。重力源＝惑星エンティティ＋小惑星クラスタ。各対象に応答係数を乗算（プレイヤー=1、敵=`def.gravityResponse`）。プレイヤーがダッシュ中は重力 ×0.25。グレネードのみ projectile として対象に含める（弾・ビーム・敵弾は対象外）。
- `MovementSystem`（プレイヤー）: ハイブリッド慣性へ（friction 調整＋ダッシュ momentum 注入）。積分後の衝突に**惑星円衝突＋クラッシュ判定**を追加。
- `AISystem`（敵）: AI 移動デルタに加えて drift（momentum）を積分（軽い減衰）。重力免疫の敵は drift に重力が積まれないので結果的に AI 純移動。惑星円衝突＋クラッシュ判定を追加。
- `WorldFactory`: 宇宙ステージ生成時に惑星エンティティを 2〜4 個配置（seed 固定）。惑星 family／holder を inject。
- 描画（`render/` ＋ `GameScreen` ＋ `SnapshotSystem`）: 惑星円（biome 色）と重力井戸リングを描画。既存の `GravityField` リング描写を惑星にも拡張。

### 4.4 システム順序
`GravitySystem`（drift へ重力を蓄積）→ プレイヤー積分（`MovementSystem`）→ 敵積分（`AISystem`）の順を推奨。drift は跨フレームで持続するため 1 フレーム遅延は許容。現在 `GravitySystem` は `MovementSystem` の後にあるため、`WorldFactory` の `systems{}` 登録順で前へ移す。

## 5. 詳細仕様

### 5.1 慣性（ハイブリッド）
- `driftX/driftY` を唯一の移動速度（momentum）とする。
- **プレイヤー**:
  - 通常移動: 入力方向へ加速度を加算、移動中 friction は中程度（上限に到達）。
  - 入力を離すと coast（滑空）。`STOP_FRICTION` を緩めて滑空を出す（初期 0.08→約 0.5、§6 で要調整）。
  - ダッシュ: drift へ強い加速を注入し、ダッシュ終了後も momentum が残る。`DASH_ACCEL` 維持＋必要なら追加注入。
- **敵**:
  - AI が算出する移動デルタは従来どおり直接位置へ（AI のキレを保つ）。
  - それとは別に drift を積分: `総移動 = AI移動 + (drift + knockback)*dt`。
  - drift は軽い減衰（吹き飛ばされた敵が滑空 → やがて AI が復帰）。
  - drift に重力・クラッシュ反発・将来の外力が積まれる。重力免疫（応答0）の敵は drift が育たず AI 純移動。

### 5.2 重力
- `combinedGravityAccel(planets, clusters, x, y)` が合算加速度 `(ax, ay)` を返す。
  - 惑星: 中心方向、質量 ∝ 引力、`gravityRange` 内で距離減衰。
  - 小惑星クラスタ: 既存 `WallGravity.gravityAt`（強さ・範囲は現行値ベース）。
- 適用: `v.driftX += ax * response * dashMul * dt`（`dashMul` はプレイヤーがダッシュ中のみ 0.25、他は 1.0）。
- 対象: プレイヤー、敵、グレネード。弾・ビーム・敵弾は対象外（視認性優先）。

### 5.3 惑星エンティティ
- `WorldFactory.create` で宇宙ステージに 2〜4 個生成。プレイヤー初期位置から一定距離を空け、互いに重ならないよう配置。半径・質量・biome を seed 乱数で決定（テスト決定性のため `Rng(seed xor ...)`）。
- 惑星は**ソリッド**（通り抜け不可）。タイルではなく円エンティティとして扱う。
- 描画: biome に応じた色の円＋重力井戸の同心リング。

### 5.4 衝突・クラッシュ
- **壁（小惑星タイル）**: `moveAndCollide` が `hitX/hitY` を返した軸で、当該軸 drift の大きさが閾値超なら `CrashModel` でダメージ。その後 drift をゼロ（現行どおり）。
- **惑星（円）**: 積分後の位置を `CircleCollision.resolve` で判定。内側なら表面へ押し出し、内向き速度を取得。内向き速度が閾値超なら `CrashModel` でダメージ＋内向き drift を相殺し軽く反発。
- クラッシュダメージはプレイヤー・敵**双方**に適用。プレイヤーには既存 i-frame を尊重。敵が壁／惑星で撃破されたら通常のキル処理（ドロップ等）に乗せる。
- `ダメージ = k * max(0, 内向き速度 - 閾値)`。

### 5.5 敵の重力応答（データ駆動）
- `EnemyDef.gravityResponse` で表現。
  - `0`: 重力無視（軽い飛行体。直進してくる脅威）。
  - `1`: 通常。
  - `>1`（例 1.5）: 重い。重力で隊列が崩しやすく、惑星へ叩きつけやすい獲物。
- 既存敵は既定 `1` のまま。数体に `0` と `1.5` を設定して非対称性を導入（具体割当は SP-5 で）。

### 5.6 弾・グレネード
- 通常弾／ビーム／敵弾: 直進（重力対象外）。
- グレネード: 重力対象（既に物理的に投擲される弾種のため、曲がりが自然）。

## 6. 既定パラメータ（初期値・要チューニング）

| 項目 | 初期値 | 備考 |
|---|---|---|
| プレイヤー coast 減衰 `STOP_FRICTION` | 約 0.5 | **最重要チューニング**。大きいほど長く滑空。実機で詰める |
| プレイヤー移動中 friction | 0.6（現状維持） | 上限到達の安定性 |
| ダッシュ momentum 注入 | `DASH_ACCEL=1600` 維持＋必要に応じ追加 | 滑空の強さ |
| 敵 drift 減衰 | 毎秒 〜20%（`0.8^dt` 目安） | 吹き飛ばし後に AI 復帰 |
| ダッシュ中の重力倍率 | ×0.25 | 重力脱出の効き |
| クラッシュ速度閾値 | 中速相当（実装時に実速度系で決定） | これ未満は無傷 |
| クラッシュダメージ係数 `k` | 調整 | 閾値超過分に比例 |
| クラッシュ反発 | 内向き相殺＋軽い反発 | 弾まない |
| 惑星数 | 2〜4 | seed 固定 |
| 惑星半径／質量 | ランダム範囲 | 実装時に範囲決定 |

数値は実機プレイで最終決定する前提。`STOP_FRICTION`（滑空の長さ）と重力強度・クラッシュ閾値が手触りの主軸。

## 7. テスト方針

純粋モジュール中心に TDD（既存 175 テストは維持）。

- `Inertia`: 無入力で drift が減衰し続ける／上限でクランプ／ダッシュ注入で drift 増加。
- `combinedGravityAccel` / `PlanetGravity`: 範囲外で 0／範囲内で中心方向／距離で減衰／質量で強さ変化／複数源の合算。
- `CircleCollision.resolve`: 円外は不変／円内は表面へ押し出し／内向き速度を正しく返す。
- `CrashModel`: 閾値以下は 0 ダメージ／超過分に比例／反発後速度の符号。
- `gravityResponse`: 0 で重力加算が 0／1 で等倍／1.5 で増。
- 統合確認（可能なら desktop launcher）: 滑空・重力での軌道・ダッシュ脱出・敵を惑星へ叩きつけ。

各フェーズで `./gradlew :core:test`、可能なら `:android:assembleDebug` を通す。

## 8. 実装フェーズ

各フェーズは「最小実装＋テスト＋`:core:test` 緑＋コミット」を満たす。

- **SP-1 慣性のハイブリッド化（プレイヤー）**

  - 完了条件: 宇宙ステージに惑星が出て重力源になり、ソリッドとして衝突する。テスト追加。
- **SP-4 クラッシュダメージ**
  - `CrashModel`。壁・惑星双方、プレイヤー＆敵。
  - 完了条件: 高速衝突でダメージ。敵を惑星／壁へ叩きつけて撃破できる。テスト追加。
- **SP-5 `gravityResponse`＋敵チューニング＋バランス**
  - `EnemyDef.gravityResponse`、`GravitySystem` で反映、敵への割当、全体チューニング。
  - 完了条件: 重力無視／重い敵が存在し非対称な立ち回りが生まれる。テスト追加。

## 9. スコープ外 / 将来接続

本設計に含めない（後続ラウンド）：
- 惑星への着陸・地表ステージ遷移・地表物理。
- biome 別の地表環境（氷で滑る等）。
- 生き物 AI（恐怖・逃走・命乞い・守護）、吹き出し・会話、部族・家族・王。
- biome 別の敵セット、惑星素材と武器派生。

本設計の `Planet` エンティティ（座標＋半径＋質量＋biome）は、将来の Living Planets で**そのまま着陸対象・バイオーム保持体**として再利用できるよう最小設計にしておく。

## 10. リスクと留意点

- **操作性**: ハイブリッドでも滑空が長すぎると敵弾回避が難しい。`STOP_FRICTION` を主軸に実機調整。
- **クラッシュの理不尽さ**: プレイヤーが事故死しないよう閾値・i-frame を慎重に。惑星配置はプレイヤー初期位置から距離を確保。
- **敵 drift と AI の競合**: AI 移動は位置直接・drift は別積分で分離し、二重計上を避ける（v2.9.0 で起きた direct+drift 二重計上の轍を踏まない）。
- **巨大マップ性能**: 重力検出は現行どおりプレイヤー周辺ウィンドウ＋スロットルを維持。惑星は数個なので影響軽微。
- **描画の決定性**: 物理ロジックは純粋・テスト可能に保ち、描画は別レイヤ。
